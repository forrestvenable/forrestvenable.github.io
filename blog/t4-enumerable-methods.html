Enumerable Methods

October 11, 2014

This week was a lot of work with the various "enumerables" methods. Enumerables are hashes and arrays, or more generally any kind of list of elements. Ruby has a wide variety of different tasks that can be done with them, such as multiplying each element by 3 or printing only even numbers. In general their format is "enumerable.method(optional parameter){|iterators| code_to_be_done}". My focus for this blog post is on the "cycle" method, but the basics are more broadly applicable

Cycle takes an array, such as [1,2,3,4]. It then goes over each element of the array a number of times, which is specified with the parentheses. If no number is specified, it will default to doing it forever, which means that if you aren't careful it won't stop and will bog things down. What it does with each iteration is specified within curly brackets and can be just about anything. If I want to multiply every number by two and repeat the array 3 times, I can write: [1,2,3,4].cycle(3){|n| puts n*2} The computer will then go over the array and print n*2 while substituting whichever current number it is on for n. It will stop the 3rd time it goes through the array. This means that the end result is the following sequence: 2, 4, 6, 8, 2, 4, 6, 8, 2, 4, 6, 8.

Note that I am not required to actually include n in the code to be executed. If I write[1,2,3].cycle(1){|n| puts "Panda"} I will print Panda three times. This means that "cycle(1)" is identical to "each", as they will both iterate over every element in an enumerable once.
