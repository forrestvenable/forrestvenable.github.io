Pair Programming

Two or more programmers working on code. Working together to not only collaborate but also split workload. This allows knowledge to be shared and increases efficiency. While one is coding, another can look up syntax specific information.

Pair Programming Important to DBC

It is important because helps people share knowledge and help develop an understanding of code standards and rationales for doing things. This also avoids the stereotype of a lone programmer hacking away at code in the night. In addition, it is good practice for real world job positions for web developers.

Driver vs. Navigator

Driver - Is the person who actually writing the code

Navigator - Is the person who organizes the driver's effort and sets priorities.
